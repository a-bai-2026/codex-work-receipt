# Changelog

## 0.10.0

- 新增 `--custom-range`、`--from` 和 `--to`，支持自定义完整自然日或精确到时分的统计区间。
- 自定义自然日继续生成 cwr2 规范事实；精确时间区间使用不参与供销社统计的兼容 cwr1 私人摘要。
- 新增 `--select-session`、`--select-project` 和 `--project <directory>`，支持交互选择会话、项目或直接按项目目录筛选。
- 项目仓库地址与路径立即转换为本机加盐的匿名身份，不写入 HTML、历史或微信导入文件。
- HTML“更多小票功能”扩展为 15 条命令，并按时间范围、会话与项目、自动模式、票仔扩展四个可访问 Tab 展示。
- 命令 Tab 记忆选择状态，支持键盘切换、桌面双列和移动端横向导航，继续排除在长图导出之外。
- 修复自动模式合并相邻 Stop 触发时丢失原始触发时间、可能导致跨午夜小票写入错误日期的问题。

## 0.9.0

- 新增缓存命中率、每轮 Token / 输出 Token / 工具调用 / 完成耗时等效率指标。
- 新增首 Token 与整轮耗时的 P50 / P90 分位数，以及按本地时区生成的 24 小时工作热力图。
- 新增模型使用轮次和脱敏工具类别结构；解析器不保留原始工具名、调用参数、命令或工具输出。
- HTML 与本地完整 JSON 展示详细效率洞察；微信 `cwr1` / `cwr2` 紧凑载荷保持不变，继续兼容现有小程序。
- HTML 赞助商下方新增“更多小票功能”，集中展示生成范围、自动 / 手动模式、Skill 与票仔的 12 条常用命令，并支持一键复制。
- 功能命令区采用独立的浅色纸张样式，记忆展开状态，适配桌面和移动端，并继续排除在长图导出范围之外。

## 0.8.0

- 首次交互式运行新增“自动保存 / 仅手动”选择；显式范围命令继续保持非交互兼容。
- 自动保存通过 Codex `Stop` Hook 静默刷新当天同一张 HTML、完整 JSON 和 `.cwr.json` 微信导入文件，不使用定时器、空闲判断或常驻监听。
- 新增 `--setup`、`--enable-auto`、`--disable-auto` 和 `--auto-status`，支持安全合并、检查和移除本工具自己的用户级 Hook。
- 自动模式安装可离线运行的稳定本地执行器，合并前备份现有 Hooks，并在并发触发时合并任务、保护同一日输出。
- 新增 `cwr-file-v1` 微信导入文件：每张小票在 HTML 旁边输出一个带完整 SHA-256 校验的 `.cwr.json`，网页也可直接下载同一份文件。
- 桌面导入流程改为“下载文件 → 发送到文件传输助手 → 小程序选择聊天文件”，并补充小程序接入规范和跨仓库兼容夹具。
- 数据二维码降级为可选单码：只有完整载荷能放进安全 QR version 时才显示扫码入口，不再生成或轮播新的 multipart 二维码。
- 保留 `cwr1`、`cwr2` 和历史 `cwr2p` 解码兼容；二维码容量不足不再阻止 HTML、JSON、导入文件或本地历史生成。
- 修复 HTML 长图导出时文字节点被固化宽高后发生的数值换行与错位问题。
- 更新 HTML 工具栏并新增响应式侧栏，改善主题切换、长图导出和相关信息浏览体验；这些网页控件不会进入导出图片。
- 更新中英文 CLI、手机导入、协议、隐私和 Codex Skill 文档。

## 0.7.4

- 修复部分 bug。

## 0.7.3

- Codex 会话日志改为分块逐行解析，避免超大 `.jsonl` 在转成单个 JavaScript 字符串时触发 V8 长度上限。
- 会话行解析后立即丢弃 Prompt、回复、图片和工具输出，只在内存中保留统计所需的时间、事件、模型与 Token 字段。
- 单行异常过大时带文件路径和行号跳过；文件读取失败时补充路径与体积，便于定位具体日志。
- 本地 `history.jsonl` 改为逐条写入，避免历史增长后形成同类的大字符串拼接风险。

## 0.7.2

- 修复同一 Codex 会话存在多份 append-only 日志修订时生成重复 canonical factId、导致小程序云端入库拒绝的问题。
- cwr2 manifest 输出前新增 factId 唯一性防线，避免无效二维码进入扫码链路。
- “最近 N 小时”在交互选择、终端输出和 HTML 中明确标记为私人滚动摘要，并提示其不参与 AI 供销社统计。

## 0.7.1

- 修正 HTML 长图导出：PNG 只保留完整小票和微信小程序码，不再包含数据二维码或分片数据码。

## 0.7.0

- 新增 Codex 原生桌宠“票仔 · AI 小票工”，提供夜班绿票风格的核心状态与左右移动动画，并支持通过 `--install-pet`、`--uninstall-pet` 和 `--install-companion` 安装管理。
- AI 打工小票 Skill 新增“票仔，开今天的票”等自然语言入口。
- 新增 `--hours <1-168>` 滚动小时统计；“最近几个小时”默认按最近 3 小时处理。
- 滚动小时小票使用兼容 cwr1 摘要，避免与 cwr2 的“会话 × 自然日”规范事实冲突。

## 0.6.1

- 将 HTML 小票底部的重复渐变装饰替换为按小票编号生成的真实 Code 128 B 条形码，并保留适合浏览器和长图导出的静区与清晰度。
- 优化 multipart 扫码交互：单数据码继续与小程序码并排，多分片时先展示小程序码，再自动逐张轮播唯一可见的数据码，避免镜头同时识别多个二维码。

## 0.6.0

- 新增 cwr2 协议：按“会话 × 记账自然日”生成稳定的 canonical facts，重叠的今日、近 7 日和本周小票可由接收端可靠去重。
- 新增 fact、session、receipt 的稳定匿名身份，以及 manifest、content hash 和 source revision，便于幂等导入和追加式会话更新。
- Token 统计改为按来源顺序处理累计值重置，避免重置后出现负数或漏计。
- 新增 `cwr2p` 分片二维码；单个 cwr2 超出安全二维码容量时自动拆分，最多 12 片，并支持乱序重组和分片/总校验。
- 网页导入联可展示多张数据二维码；命令行输出二维码数量和 canonical fact 数量。
- 保留 cwr1 解码兼容；旧版小票仍可作为私人历史导入，但不包含精确去重所需的 canonical facts。
- 更新中英文手机导入、数据协议和隐私说明。

## 0.5.0

- 新增网页版“保存完整长图”功能。
- 高清 PNG 包含主小票、小程序码和数据二维码导入联。
- 导出图片自动使用当前主题，并排除主题按钮、网页背景和其他页面元素。
- 网页主题与小程序统一为经典白票、复古粉票和夜班绿票。
- 小票顶部明确展示统计日期范围，默认文件名同步携带日期范围或会话短编号。
- 图片在浏览器本地渲染，不上传小票数据。
